Monolith — animations "assistant"

Prototypes CSS + SVG sur la piste 04 (le logo retenu). Référence vivante des pistes d'animation "assistant" pour le Monolith, à faire évoluer avant de choisir ce qui sera branché dans l'app.

Ce fichier est une référence visuelle versionnée — 100 % autonome (SVG inline + CSS @keyframes, zéro dépendance externe), ouvrable directement au navigateur (xdg-open docs/design/monolith-animations.html). Les 3 expressions produites par le script Python (default, neutral, wink) sont réutilisées ici, et de nouvelles pistes (smile, shake, mouth-talk, scan-line, boot, dots, ring, bar, typing, chevron rain) sont composées directement en CSS pour l'exploration — aucune n'est encore exportée en asset ni branchée côté app.

Note perf — toutes ces anims sont du CSS @keyframes pur, rendues par le compositor du navigateur. Une fois branchées côté app (GPUI ne rejoue pas les <style> inline dans un SVG chargé via img()), il faudra probablement passer par un état Rust qui bascule entre plusieurs assets SVG statiques (comme le fait déjà le générateur pour default/neutral/wink), ou animer via with_animation côté GPUI en swap de source. C'est une décision d'implémentation à prendre après avoir choisi ce qu'on veut animer.
