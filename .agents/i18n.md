# Internationalisation (i18n)

ShellDeck utilise **rust-i18n** pour tous les textes visibles de l'interface.
Ne plus ajouter de chaînes UI en dur dans les vues.

## Langue par défaut : **français**

- Locale rust-i18n par défaut et **fallback** : `fr`.
- `GeneralConfig.ui_language` par défaut : `System` → si la locale OS n'est
  pas reconnaissable, retomber sur **`fr`** (pas `en`).
- Les clés manquantes en traduction doivent retomber sur le français, pas
  l'anglais.
- Nouveaux écrans : rédiger la clé source en **français** dans
  `locales/fr.toml`, puis ajouter la traduction anglaise dans
  `locales/en.toml`.

## Fichiers

```
crates/shelldeck-core/locales/   # fr.toml + en.toml (canonique)
crates/shelldeck-ui/src/i18n.rs  # apply_ui_language, rel_time
crates/shelldeck-ui/src/lib.rs   # rust_i18n::i18n!(…) + macro t!
```

Métadonnées : `[package.metadata.i18n]` dans `shelldeck-ui/Cargo.toml`
(`default-locale = "fr"`, `load-path = "../shelldeck-core/locales"`).

Préférence persistée : `[general] ui_language` dans `shelldeck.toml`
(`UiLanguage` dans `shelldeck-core`).

## Usage dans une vue (`shelldeck-ui`)

```rust
use crate::t;

.child(t!("login.title"))
.child(t!("login.device", device = self.device.clone()))
```

- Clés en **snake_case** avec préfixe module : `login.submit`, `settings.tab.general`.
- Variables : syntaxe `%{name}` dans le TOML, `name = value` dans `t!`.
- Brand names / noms propres (Google, GitHub, Inklura Manage) : laisser tels
  quels dans les deux locales sauf phrase entière à traduire.

## Changement de langue

- Appliquer via `shelldeck_ui::i18n::apply_ui_language(&config.general.ui_language)`
  au démarrage (`main.rs`) et sur `SettingsEvent::ConfigChanged` (workspace).
- Après `set_locale`, **`cx.notify()`** pour repaint.

## Migration progressive

1. Extraire les chaînes de l'écran touché vers `fr.toml` + `en.toml`.
2. Remplacer les literals par `t!(…)`.
3. Ne pas mélanger FR/EN dans le même fichier sans clé i18n.
4. `cargo i18n` (CLI optionnelle, crate `shelldeck-ui`) pour les clés manquantes.

## Dates relatives

Utiliser `crate::i18n::rel_time(at_ms)` — ne pas dupliquer
`"à l'instant"` / `"il y a …"` dans les vues.

## Hors scope i18n

- Sortie terminal SSH, logs, messages d'erreur réseau bruts.
- Contenu utilisateur (tickets support, scripts, noms de connexion).
- `CalendarLocale` adabraka-ui : reste local au composant calendrier.
